## Column Descriptions

- **CIP Number**: Link to the CIP in the [CIP repository](https://github.com/cardano-foundation/CIPs)
- **Title**: Full title of the CIP
- **Category**: Ledger | Consensus | Plutus | Networking | Cross-cutting
- **Hard fork Required**: Yes | No | Unknown
- **Need new Ledger era?**: Yes | No | Unknown
- **Status**: Proposed | Under Review | Approved for Implementation | In Progress | Implemented
- **Priority**: High | Medium | Low (updated during workshops)
- **Rank**: Numerical rank (1-N, updated quarterly)
- **Dependencies**: Links to other CIPs this depends on (if any)
- **Impact Areas**: Which node components are affected
- **Added Date**: Date when CIP was added to shortlist (YYYY-MM-DD)
- **Last Reviewed**: Date of last workshop review (YYYY-MM-DD)
- **Notes**: Brief rationale or key considerations

## Adding a CIP

**For Proposers**: To propose a CIP for the shortlist, create a new file `CIP-XXXX.md` in this directory following the guidelines in [CONTRIBUTING.md](../CONTRIBUTING.md). Do not edit this table directly.

**For Maintainers**: After a CIP detail file PR is merged, add an entry to this table with information from the detail file. The table provides a quick overview, while the detail files contain comprehensive information.
